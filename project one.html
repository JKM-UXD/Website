The project was created within the first half of two courses on interaction design. During the 9 weeks we made use of several techniques to create a entirely new website wireframe as part of a complete digital system for the fictional company UK cinemas (Bios) in the Netherlands. Our aim was to create an all-in-one group communication tool that would provide several ways to organise and discuss visits, vote for movies to watch and split the payment among the group members. Our target group were digital natives.

After creating a design brief conducted first analysis of user needs and user stories as well as doing a task analysis. The task analysis consisted of defining actors & use cases, visualising the relation in a storyboard and creating a hierarchical task diagram. We then went to the ideation phase applying several techniques like formulating HMW questions, doing a 6-3-5 brainwriting session and organising the ideas in an affinity diagram. After evaluation of the ideas, we created three value propositions from which we would choose one to implement. Our value proposition would implement an all-in-one group communication tool that would provide several ways to organise and discuss visits, vote for movies to watch and split the payment among the group members. Therefore, groups would not have to face the usual issues of failed organisation and debts which would drastically improve the overall experience of going to the cinema. We elaborated this value proposition with a concrete scenario with a story. Furthermore, we defined a use case description and formulated the requirements for our system. Our next step was to do a card-sorting session with several users. From the results we did a taxonomy to define the categorisation of movies and genres. We also created a flow chart to get a clear overview of the user flow in our system. In week 6 we went on creating a complete sitemap. Moreover, we went on creating a first iteration of the website as a lo-fi wireframe. In the following week we translated our very basic wireframe into a mid-fi wireframe that would (visually) represent our concept of the group communication tool. We gathered feedback and implemented these improvements into our final version. I worked on the final version and also implemented several steps and screens which we hadn’t thought about in the previous versions (like the checkout). Moreover, I went on to prototype the entire wireframe also implementing logic into the system. Furthermore, I added multiple interactive elements and screens. Lastly, we created a few placeholders which would contribute to the impression of an actual system.
